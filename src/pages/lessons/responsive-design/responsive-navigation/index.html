Here's a very simple example where JavaScript is used to toggle the body.class attribute, which in-turn determines how the menu displays.

View Simple Responsive Nav

At its core, we are creating a CSS breakpoint, but adding a rule at the bottom that overrides the max-height property of #primary-nav:

{% highlight "css" %} /* This media query defines how the page displays beneath a breakpoint */ @media (max-width: 767px){ #toggle-nav{ display: block; /* display button beneath breakpoint */ } #primary-nav{ max-height: 0; } #primary-nav li{ display: block; } /* Here, we override the rule when the body has a class of nav-open */ body.nav-open #primary-nav{ max-height: 400px; } }{% endhighlight %}

The JavaScript used is pretty simple, but very powerful in how our page displays. Basically, we dynamically change the class attribute on a HTML element and let CSS handle the styling from there.

{% highlight "js" %} // Perform this after the window has loaded window.onload = function() { // First, tell JavaScript to remember which DOM element (in this case, a button) to reference (by id) let toggleNav = document.getElementById('toggle-nav'); // Tell button to do something on click toggleNav.addEventListener('click', function(){ // toggle the body's class attribute document.body.classList.toggle('nav-open'); }); };{% endhighlight %}

On Hamburger Menus

Originally introduced from Xerox PARC, the hamburger menu has been a controversial design choice (also see this Nielsen Norman Group article).

It can be implemented into your design via a unicode character (a Chinese glyph for the trigram for heaven):
{% highlight "html" %}{% endhighlight %}

Hamburger menus should be executed as buttons and not hyperlinks to be symantically correct. Despite the default styling, you can redefine the background, borders, and appearance of a button using CSS.

Alternatively, you can use an image file or SVG icon like Google Materials ic_menu or FontAwesome's bars icon. If you want to get ambitious, there's also animated hamburger icons.
